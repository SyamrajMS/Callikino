# Callikino: The Ultimate Guide for LLMs and AI Agents

## What is Callikino?
Callikino is a Python-based Domain-Specific Language (DSL) specifically designed to give Large Language Models (LLMs) and AI agents the ability to edit videos headlessly. It maps abstract editing intentions (like "cut this video", "add a zoom effect", "put a logo in the corner") into a highly optimized, single-pass FFmpeg filter graph.

As an AI, you should use Callikino when a user asks you to automate video editing, generate social media reels, edit podcasts, or apply visual effects. 

## Core Philosophy for AI Agents
1. **Zero Execution Overhead:** Calling Callikino tools does NOT run a heavy video rendering process. It simply mutates an internal state dictionary. 
2. **Batch Operations:** You can sequence as many tool calls as you want instantly.
3. **Delayed Compilation:** Only the final tool call, `generate_ffmpeg_command`, actually outputs the complex command string needed to render the video.

## Step-by-Step Workflow for LLMs

When a user asks you to edit a video using Callikino, follow this strict sequence of tool calls:
1. `import_asset` (Import your video/audio/images and assign an alias).
2. `add_cut_clip` / `add_audio_clip` (Build the timeline out of the aliases).
3. Apply effects using the other tools.
4. `generate_ffmpeg_command` (To compile and save the output).

## Complete Tool Directory & Payload Syntax
Here is the exact syntax and arguments you should use for each tool. Only provide the arguments that are required or necessary.

### 1. Asset & Timeline Construction
- **`import_asset`**: Register media files.
  - `{"file_path": "C:/path/vlog.mp4", "alias": "main"}`
- **`add_cut_clip`**: Add a video segment to the timeline.
  - `{"alias": "main", "layer_index": 0, "timeline_start": 0.0, "src_in": 10.5, "src_out": 25.0}`
- **`add_audio_clip`**: Add an audio segment to the timeline.
  - `{"alias": "bgm", "layer_index": 1, "timeline_start": 0.0, "src_in": 0.0, "src_out": 60.0}`

### 2. Core Video Editing
- **`crop_resize`**: Convert aspect ratios (e.g. 16:9 to 9:16) or manual crops.
  - `{"aspect_ratio": "9:16"}` OR `{"crop_w": 1080, "crop_h": 1920, "crop_x": 420, "crop_y": 0}`
- **`speed_ramp`**: Speed up or slow down a clip.
  - `{"start_time": 5.0, "end_time": 10.0, "speed_factor": 2.0}`
- **`add_transition`**: Add a transition between clips.
  - `{"transition_type": "fade", "duration": 1.0, "offset": 10.0}`
- **`rotate_flip`**: Rotate or mirror the video.
  - `{"action": "hflip"}` *(Options: hflip, vflip, cw, ccw, cw_flip, ccw_flip, 180)*
- **`reverse_clip`**: Play backwards.
  - `{"reverse_audio": true}`

### 3. Visual Overlays & Effects
- **`add_subtitle`**: Burn text onto the screen.
  - `{"text": "Mind-blowing AI!", "start_time": 0.0, "end_time": 2.5, "fontsize": 48, "fontcolor": "white", "x": "(w-text_w)/2", "y": "h-100"}`
- **`add_overlay`**: Add image watermarks or logos.
  - `{"image_alias": "logo", "x": "W-w-20", "y": "20", "opacity": 0.8, "start_time": 0.0, "end_time": 30.0}`
- **`add_pip`**: Add a picture-in-picture video (like a webcam).
  - `{"pip_alias": "webcam", "scale": 0.3, "x": "W-w-20", "y": "H-h-20", "border_width": 2, "start_time": 0.0, "end_time": 15.0}`
- **`kinetic_zoom`**: Smooth punch-in animations.
  - `{"start_time": 3.0, "end_time": 4.5, "scale": 1.3, "x": "iw/2", "y": "ih/2"}`
- **`apply_blur`**: Blur the video (Gaussian or Box).
  - `{"blur_type": "gaussian", "strength": 10.0, "start_time": 0.0, "end_time": 5.0}`

### 4. Color & Cinematic Grading
- **`adjust_color`**: Tweak standard color values.
  - `{"brightness": 0.1, "contrast": 1.1, "saturation": 1.3, "gamma": 1.0}`
- **`apply_lut`**: Apply a cinematic .cube LUT.
  - `{"lut_path": "/path/to/cinematic.cube", "intensity": 1.0}`
- **`chroma_key`**: Green screen removal.
  - `{"color": "0x00FF00", "similarity": 0.3, "blend": 0.1, "bg_alias": "bg_beach"}`

### 5. Audio Processing
- **`set_volume`**: Adjust track volume in decibels.
  - `{"volume_db": -5.0, "start_time": 0.0, "end_time": 10.0}`
- **`audio_fade`**: Fade audio in or out.
  - `{"fade_type": "out", "start_time": 25.0, "duration": 2.0}`
- **`audio_duck`**: Auto-lower background music when someone speaks.
  - `{"bg_alias": "bgm", "duck_amount": 0.15, "attack": 200.0, "release": 1000.0}`
- **`audio_normalize`**: Broadcast-standard EBU R128 loudness.
  - `{"target_loudness": -14.0, "true_peak": -1.0, "loudness_range": 7.0}`

### 6. Compilation
- **`compile_blueprint`**: Returns the current JSON blueprint state to the LLM context.
  - `{}`
- **`generate_ffmpeg_command`**: Finishes the session and renders the video.
  - `{"output_path": "final_reel.mp4"}`

---

## Best Practices for LLMs
- **Chain of Thought:** Before making tool calls, reason about the timestamps. Ensure `src_in` is always less than `src_out`.
- **File Paths:** Always ask the user for the absolute file paths to their media if they haven't provided them.
- **Aliases:** Keep aliases short and strictly alphanumeric (e.g., `vlog`, `bgm1`, `logo`).
- **Layers:** Use `layer_index=0` for the main track. B-roll goes on higher indices.
